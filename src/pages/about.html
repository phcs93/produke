proDuke is a Duke Nukem 3D source port that primarily focuses on online matches. We consider it very important to keep the core gameplay intact; however, at the same time, and with the intention of providing gameplay variety, we also place equal importance on adding common features from other source ports of classic games (e.g., zDaemon in DOOM). These are always implemented in an optional way, allowing the base game to be played identically to how it has always been played, if that is the player's intention.

proDuke is based on xDuke 19.7.1, due to its gameplay being closer to the original. EDuke32, although it has many qualities, also contains numerous changes that make it a non-ideal option for this project's goals.

Currently, the project's source code is closed. This is because the game features extremely intense gameplay, which creates a highly competitive environment and ego clashes. As a result, attempts to use cheats are constant, and releasing the project's source code would go against its main purpose, since it would enable (or greatly facilitate) the creation of cheats.

proDuke can be considered the successor of nDuke. nDuke, also coded by me, was the first source port to feature a fully functional CTF mode, including team-specific start points (both of which had never been seen in any other source port). Unfortunately, due to a hard drive failure and the simultaneous loss of backups, its source code was lost. proDuke is essentially a reimplementation and continuation of nDuke.

When nDuke was created, the project had two main motivations (which remain in proDuke):

1. To implement CTF mode. Duke's gameplay, more than any other game of its era, revolves around extremely refined movement, and logically, CTF mode fits perfectly with its mechanics.

2. To provide a solution to the toxic culture of “soft cheats” present in the community. Not everyone knows this, but it was always common for players to use modified game files to gain advantages over others, and this was never blocked in the original game (I attribute this to a critical mistake in the implementation of mod file loading, which stems either from Ken Silverman - creator of the Build engine - or from 3D Realms itself). There are ports that partially block external files, but modified .art file cheats remain allowed, and nDuke was created to put an end to that.
