To create maps using team spawn points, all you have to do is set the Hi-Tag of the APLAYER (picnum #1405) sprites to a team number. Para criar mapas usando os pontos de nascimento de times, basta setar a Hi-Tag da sprite APLAYER (picnum #1405) para o número de um time. The player one spawn point (the "arrow" on the 2d map - set with pressing the scroll-lock key in mapster32) is not used in team based modes unless there are not enough team spawn points defined. O ponto de nascimento do player um (a "flecha" na visão 2d - setado apertando a tecla scoll-lock no mapster32) não é utilizado em modos baseados em times a não ser que não existam pontos de nascimento de time o suficiente. In case the match is being played with the 
Netplay Options > NetFlags > [TEAM] Teamless Spawn Points
netflag, the game will treat the team spawn points as normal spawn points (i.e. the spawn points will be assigned to the players regardless of the teams they're in).
This useful so CTF made maps that have symmetrical bases (with corresponding team spawn points) can be played as TDM/TLMS maps, in case it's the intention to have the the players to be randomly spawned through all the map instead of spawning only on their corresponding base. It's recommended to keep this option off in case of CTF matches, because CTF maps are normally made of symmetrical bases.
Finally, if that netflag is ON and the current game mode isn't COOP/SURVIVAL, there's a special case in which proDuke will ignore the "arrow" spawn point: if the map has TEAM spawn points (i.e. APLAYER sprite with hitag > 0) BUT has no DM spawn points (i.e. APLAYER sprite with lotag = 0 and hitag = 1).
This is done because, on that case, the map has clearly been made with symmetrical bases in mind, so it makes no sense for the PVP spawns to not also be symmetrical. Note that maps that have both TEAM and normal DM spawn points will not be treated as a special case and will function normally. Caso a partida esteja sendo jogada com a netflag 
Netplay Options > NetFlags > [TEAM] Teamless Spawn Points
, o jogo irá tratar os pontos de nascimento de time como pontos de nascimento normais (i.e. os pontos do nascimento serão distribuídos entre os players independente dos times a qual façam parte).
Esse comportamento é recomendado para que mapas feitos para CTF com bases simétricas (com os pontos de nascimento correspondentes) possam ser jogados como mapas de TDM/TLMS, caso não seja a intenção de os players nascerem aleatoriamente por todo o mapa ao invés de apenas em suas respectivas bases. É recomendando que essa opção permaneça desligada em partidas CTF, dado que nesse modo os mapas normalmente são compostospor bases simétricas.
Por fim, se essa netflag estiver ligada e o modo de jogo atual for COOP/SOBREVIVÊNCIA, haverá um caso espeial no qual o proDuke irá ignorar o ponto de nascimento da "flecha": se o mapa tiver pontos de nascimento de TIME (i.e. sprites APLAYER com hitag > 0) MAS não tiver pontos de nascimento deDM (i.e. sprites APLAYER com lotag = 0 e hitag = 0).
Isso é feito pois, nesse caso, o mapa foi claramente feito pensando em bases simétricas, então não faria sentido para os pontos de nascimento de PVP também não serem simétricos. Note que mapas que tenham ambos pontos de nascimento de TIME e de DM não serão tratados como casos especials e functionarão normalmente.
